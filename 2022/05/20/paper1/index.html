文献详读(一)：使用监管下的机器学习技术利用虚假反馈检测增强云的可信度

发表于 2022-05-20 更新于 2022-06-19 分类于 文献详读
本文字数： 1.6k 阅读时长 ≈ 1 分钟

Fake Feedback Detection to Enhance Trust in Cloud Using Supervised Machine Learning Techniques

作者：Harsh Taneja, etc.
期刊：Proceedings of Data Analytics and Management.
年份：2022.

DOI网址：https://doi.org/10.1007/978-981-16-6285-0_61


Abstract

越来越多的公司开始将服务器上云，各种类型、体量的组织和工厂也在很多方面采用云服务，比如软件的研发、产品研发、数据的备份、邮件服务或其他许多的任务。即使使用云服务器有很多很多的好处，比如可以快速扩大规模，及时储存数据，降低总体的损耗。但是云计算在隐私方面还有一些主要的信任问题，比如，用户对透明度控制的缺失，这样就让用户不能很好的控制管理他们自己的数据。因此缺少的可信度，对于云服务器的提供者来说，和这些云架构的使用者来说都是一个特别大的议题。这篇文章聚焦于通过基于CloudArmour的信任反馈数据集（Trust Feedback Dataset）找出虚假反馈，来加强云服务器的信任管理系统。

在后续处理和数据清除中，许多的模块，比如支持向量机（Support vector machine, SVM）、朴素贝叶斯（Naive Bayes）、随机森林（Random Forest）和决策树（Decision Tree）都会在我们的数据集中被训练以寻找更精准的模块。这些全部被提出来的模型中，都展现出最好的算法，并且体现了它们的精准性，它们的准确率和召回率都分别达到了93.55%、92.90%和92.82%。

关键词：信任管理，虚假反馈监测，云服务器提供商，云服务器顾客


Issue

本文提供一种整合的分类器（Classifier）模型，使用CloudArmour数据集识别由云计算顾客给云计算提供商的虚假反馈。这种模型跟其他传统模型相比具有更好的精确度。


Method

分为2个部分：数据集和其实施

1．数据集
Cloud Armor数据集由文献[16]收集，通过开发一个云服务器爬虫收集了网上可用的云服务。通过分析，数据集收集了近6000个云服务器的元数据。通过收集云用户的反馈来创造Cloud Armor的信任反馈数据集。Cloud Armor收集了多于10000个由7000个用户发送给113个云服务提供商的反馈数据。

2．实施
先对数据进行预处理和清理，让整个数据集达到均匀性。然后在“时间戳”列中检查数据类型是字符串还是日期-时间格式，都转换为日期-时间格式，并且删掉不一致的时间戳列。
删除“NULL”的数据，把数据集的列分为“响应时间”、“速度”、“存储空间”、“特征”、“顾客服务”、“专业评级”、“可接入率”、“使用体验”和“安全性”。然后分析了“信任结果”列，对信任达到一定阈值的创建值为“1”的新布尔列“目标”（Target），否则值为“0”。再给每个元组一个独特的云服务器数字序号，以更方便后面进行不同组的比较和利用。
提出了简单的投票集成技术，使用支持向量机、朴素贝叶斯、随机森林和决策树对机器进行了训练。
投票集成公式如下，X为整合模型的值，CM是分类模型，m是用于数据分类的CM的最大数量。
$$
X=max \sum_{j=1}^m CM_j
$$
所提出的算法如下：
（1）使用如下算法将数据分成“1”和“0”两类：SVM_linear，SVM_Radial，朴素贝叶斯，随机森林和决策树。
（2）编译上面提到的模型的分类数值，使用上面的公式形成模型。
（3）使用简单的整合技术计算每种情况的值。
（4）整合模型的结果和上述传统模型进行比较。


Result

得出各模型的准确度






Conclusion

本文提出的整合模型比其他传统模型评估虚假反馈更精准。
